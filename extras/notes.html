SRAM Study Notes

1. Introduction and Core Characteristics

  • Volatile Memory: Memory contents are lost when power is removed.
  • Cost: Generally more expensive than Flash memory.
  • Speed: Significantly faster than Flash memory.
  • Purpose: Primarily used to store variables and temporary data during program execution.
  • Access Granularity: Allows access to individual bytes, meaning large chunk erasures are not required.

2. SRAM in the AVR ATtiny1626 Microcontroller

The ATtiny1626 microcontroller, as a key feature, includes SRAM for data storage:

  • Capacity:
    • The microcontroller features a 2KB SRAM, specifically mentioned for storing data in memory.
    • Note: The ATtiny1626 memory map (Figure 28) indicates a 3KB SRAM block. This suggests there might be different configurations or a slightly larger usable space than the general feature description.

3. Memory Access and Architecture

3.1 Memory-Mapped I/O (MMIO)

  • On the AVR Core, both I/O and SRAM are accessed through the data space.
  • This means SRAM can be directly accessed using standard instructions for reading and writing to memory.
  • This approach simplifies the design of the AVR Core and reduces overall chip complexity.

3.2 SRAM Address Range (ATtiny1626)

  • According to the ATtiny1626 memory map, SRAM is located in the data space from address 0x0040 to 0x107F.

3.3 Instructions for Memory Access

The following instructions are used to access memory within the data space, which includes SRAM:

  • lds (Load Direct from Data Space to Register)
  • sts (Store Direct from Register to Data Space)
  • ld (Load Indirect from Data Space to Register) - Suitable for accessing data structures like arrays using pointer registers (X, Y, Z).
  • st (Store Indirect from Register to Data Space) - Also uses pointer registers for data structure manipulation.
  • push / pop: These are stack operations that occur in SRAM.
    • Stack operations using push/pop are noted to start at address 0x3800 within SRAM.
    • Note: This address (0x3800) falls within a "Reserved" block (0x3400-0x3FFF) in the general ATtiny1626 data space map, suggesting that this reserved space is utilized for the stack.

4. Comparison with Flash Memory

  • Speed: SRAM is significantly faster.
  • Volatility: SRAM is volatile (lost on power loss), while Flash is non-volatile (retains data without power).
  • Cost: SRAM is more expensive per bit.
  • Usage: SRAM is for dynamic, temporary data (variables), whereas Flash is for static program data and can only be erased in large chunks.
